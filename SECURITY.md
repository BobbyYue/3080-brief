# Security Policy

## Reporting a vulnerability

Do not open a public issue for vulnerabilities, exposed credentials, unsafe installation behavior, or source-document privacy leaks. Use GitHub's private vulnerability reporting feature under the repository **Security** tab, or email `yuejunji2026@gmail.com`.

Include the affected version, reproduction steps using synthetic data, impact, and any proposed mitigation. Do not include real document tokens, tenant identifiers, access tokens, client secrets, or internal source content.

## Supported versions

Security fixes target the latest released version. Older versions may be asked to upgrade before a fix is evaluated.

## Scope

Relevant reports include dependency-install approval bypasses, command injection, unsafe path handling, credential disclosure, unintended source modification, and publication of private source material.
